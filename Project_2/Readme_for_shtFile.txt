ΠΑΡΑΔΟΧΕΣ
1) 1 bucket αντιστοιχεί σε 1 block
2) H create_index λειτουργεί αρχικά με βάθος 2 λόγω της main αλλά μπορεί να λειτουργήσει μέχρι και με βάθος 7 αν αυτό αλλαχτεί από την main.
3) Λόγω του ότι οι πόλεις δεν είναι απεριόριστες όπως τα ids, η δομή μπορεί να δεχτεί περιορισμένο αριθμό εγγραφών ο οποίος εξαρτάται
και από τις τιμές που δίνει η rand().

Η οργάνωση των block έχει γίνει ακριβώς όπως στην πρώτη εργασία με μία εξαίρεση που περιγράφεται παρακάτω.
Για να διαβάσετε την λογική της πρώτης εργασίας μπορείτε να δείτε το αντίστοιχο readme που επισυνάπτεται.

Η εξαίρεση είναι η εξής:
Στο πρώτο block του δευτερεύοντος αρχείου αποθηκεύεται το filename, το attr name και το attr length.
Αυτό έχει ως αποτέλεσμα όλες οι υπόλοιπες πληροφορίες που υπήρχαν στην πρώτη εργασία να πηγαίνουν ένα block πιο μετά.


Το tupleid το έχουμε υλοποιήσει ως struct, το οποίο περιέχει ένα int που αντιστοιχεί στο block και ένα int που αντιστοιχεί στην θέση.

Η συνάρτηση SHT_SecondaryUpdateEntry() δέχεται σαν όρισμα τον πίνακα updateArray που είναι ο πίνακας που περιέχει τις αλλαγές που έχουν
προκύψει μέσα στο ίδιο ή διαφορετικό block εξετίας του πρωτεύοντος ευρετηρίου και οι οποίες πρέπει να ενημερωθούν και στο δευτερέυον.

Η συνάρτηση αυτή απο το tupleid_old βρίσκει τις παλιές θέσεις των εγγραφών που πρέπει να ενημερωθούν και τις ενημερώνει.

Στην συνάρτηση SHT_InnerJoin() πραγματοποιείται ζεύξη ανάμεσα σε δύο δευτερεύοντα ευρετήρια με ένα συγκεκριμένο κλειδί.
Έχει υλοποιηθεί ο αλγόριθμος που δόθηκε και στις διαλέξεις του μαθήματος. Πιο συγκεκριμένα διατρέχονται τα δύο αρχεία για να βρεθούν
οι εγγραφές που είναι συμβατλες με το κλειδί που δίνεται και εκτυπώνονται οι διάφοροι συνδιασμοί μεταξύ τους. 

Έχουμε φτιάξει μία main που φτιάχνει δύο πρωτεύοντα ευρετήρια και δύο δευτερεύοντα με κλειδί την πόλη και πάνω σε αυτά ελέγχονται οι διάφορες λειτουργίες.
Σαν πόλεις έχουν δοθεί συμβολοσειρές που αποτελούνται απο το γράμμα Τ και έναν τυχαίο αριθμό.
Το υλοποιήσαμε έτσι ώστε να υπάρχουν περισσότερες πόλεις από αυτές που είχαν δωθεί και να μπορεί να υπάρχει μεγαλύτερος αριθμός εγγραφών.


Έχουμε επεκτείνει τις δυνατότητες του MakeFile έτσι ώστε:
    Με την εντολή make run να μετταγλωτίζεται και να τρέχει το πρόγραμμά μας.
    Με την εντολή make ht να μετταγλωτίζεται το sht_file.c
    Με την εντολή make bf να μετταγλωτίζεται το bf.c
    Με την εντολή make clean να διαγράφονται τα περιττά αρχεία (αυτή η εντολή πρέπει να εκτελείται πριν ξανατρέξουμε το πρόγραμμά).
